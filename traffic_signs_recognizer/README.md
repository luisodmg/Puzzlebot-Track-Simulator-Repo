# Traffic Sign Detection — Project Workspace

Custom YOLOv8 training pipeline for 4 traffic sign classes used with the TE3002B simulator.

---

## Project Structure

```
traffic_sign_project/
│
├── dataset/                   ← Your labelled data goes here
│   ├── images/
│   │   ├── train/             ← 80% of images
│   │   └── val/               ← 20% of images
│   └── labels/
│       ├── train/             ← YOLO .txt label files (auto-generated by LabelImg/Roboflow)
│       └── val/
│
├── weights/                   ← Drop your trained best.pt here
│
├── runs/                      ← Auto-generated by train.py
│
├── dataset.yaml               ← Class names + paths config
├── classes.txt                ← LabelImg predefined class list
├── requirements.txt           ← Python dependencies
│
├── capture_dataset.py         ← Step 1: Capture images from simulator
├── train.py                   ← Step 2: Fine-tune YOLOv8
├── validate.py                ← Step 3: Check metrics (mAP, precision, recall)
├── detector.py                ← Core detection class (imported by main.py)
└── main.py                    ← Step 4: Run the robot with the trained model
```

---

## Workflow

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Collect images from the simulator
```bash
# Capture ~100–200 images per class (switch scenes/angles between saves)
python capture_dataset.py --class Stop --split train
python capture_dataset.py --class Workers --split train
python capture_dataset.py --class Go_Straight --split train
python capture_dataset.py --class Turn_Left_Right --split train

# Capture ~20% into val as well
python capture_dataset.py --class Stop --split val
# ... repeat for all classes
```

### 3. Label your images
Open **LabelImg** with `classes.txt` as predefined classes, and save labels in YOLO format.

```bash
pip install labelImg
labelImg dataset/images/train classes.txt dataset/labels/train
labelImg dataset/images/val classes.txt dataset/labels/val
```

Or use **[Roboflow](https://roboflow.com)** (free) and export as YOLO format.

Note: capture uses `Go_Straight` and `Turn_Left_Right` in file names, while labels shown in the UI come from `classes.txt`.

### 4. Train
```bash
python train.py
```
Best weights → `runs/detect/traffic_signs/weights/best.pt`

### 5. Copy weights
```bash
# Windows (PowerShell)
Copy-Item runs/detect/traffic_signs/weights/best.pt weights/best.pt -Force

# macOS / Linux
cp runs/detect/traffic_signs/weights/best.pt weights/best.pt
```

### 6. Validate
```bash
python validate.py
```
Target: **mAP50 > 0.80** before deploying.

### 7. Run the robot
```bash
python main.py
```

## Class → Robot Action Map

| Detected Sign   | Robot Behaviour           |
|-----------------|---------------------------|
| Stop            | Full stop (twist = 0)     |
| Go Straight     | Move forward (lx = 0.2)   |
| Turn Left/Right | Forward + left turn       |
| Workers         | Slow down (lx = 0.1)      |
| None / Blurry   | Stop (safe default)       |

Edit `_decide_action()` in `main.py` to change behaviour.

---

## Tips

- **Capture images from the simulator** — domain gap is the #1 cause of poor real-world performance.
- Aim for **100–200 images per class** minimum; 300–500 is better.
- Vary **angle, distance, and lighting** when capturing.
- If mAP50 is below 0.7 after training, try `yolov8s.pt` as the base model in `train.py`.
- Lower `CONF_THRESHOLD` in `detector.py` if signs are missed; raise it to reduce false positives.
